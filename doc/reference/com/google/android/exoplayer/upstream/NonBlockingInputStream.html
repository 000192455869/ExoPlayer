NonBlockingInputStream

com.google.android.exoplayer.upstream.NonBlockingInputStream
Known Indirect Subclasses
ByteArrayNonBlockingInputStream, DataSourceStream
ByteArrayNonBlockingInputStream An implementation of NonBlockingInputStream for reading data from a byte array. 
DataSourceStream Loads data from a DataSource into an in-memory Allocation. 

Class Overview

Represents a source of bytes that can be consumed by downstream components.

The read and skip methods are non-blocking, and hence return 0 (indicating that no data has been read) in the case that data is not yet available to be consumed.

Public Methods

public abstract void close ()

Closes the input stream.

public abstract long getAvailableByteCount ()

Returns the number of bytes currently available for reading or skipping. Calls to the read() and skip() methods are guaranteed to be satisfied in full if they request less than or equal to the value returned.

Returns
  • The number of bytes currently available.

public abstract boolean isEndOfStream ()

Whether the end of the data has been reached.

Returns
  • True if the end of the data has been reached, false otherwise.

public abstract int read (ByteBuffer buffer, int length)

Reads up to length bytes of data and stores them into buffer. This method may read fewer bytes, possibly 0.

Parameters
buffer The buffer into which the read data should be stored.
length The maximum number of bytes to read.
Returns
  • The actual number of bytes read, or -1 if the end of the data is reached.

public abstract int read (byte[] buffer, int offset, int length)

Reads up to length bytes of data and stores them into buffer, starting at index offset. This method may read fewer bytes, possibly 0.

Parameters
buffer The buffer into which the read data should be stored.
offset The start offset into buffer at which data should be written.
length The maximum number of bytes to read.
Returns
  • The actual number of bytes read, or -1 if the end of the data is reached.

public abstract int skip (int length)

Skips over and discards up to length bytes of data. This method may skip over some smaller number of bytes, possibly 0.

Parameters
length The maximum number of bytes to skip.
Returns
  • The actual number of bytes skipped, or -1 if the end of the data is reached.
