Class Overview

Facilitates the extraction of data from the fragmented mp4 container format.

This implementation only supports de-muxed (i.e. single track) streams.

public static final int WORKAROUND_EVERY_VIDEO_FRAME_IS_SYNC_FRAME

Flag to work around an issue in some video streams where every frame is marked as a sync frame. The workaround overrides the sync frame flags in the stream, forcing them to false except for the first sample in each segment.

This flag does nothing if the stream is not a video stream.

Constant Value: 1 (0x00000001)

public int read (NonBlockingInputStream inputStream, SampleHolder out)

Consumes data from a NonBlockingInputStream.

The read terminates if the end of the input stream is reached, if an attempt to read from the input stream returned 0 bytes of data, or if a sample is read. The returned flags indicate both the reason for termination and data that was parsed during the read.

If the returned flags include RESULT_READ_SAMPLE_PARTIAL then the sample has been partially read into out. Hence the same SampleHolder instance must be passed in subsequent calls until the whole sample has been read.

Parameters
inputStream The input stream from which data should be read.
out A SampleHolder into which the next sample should be read. If null then RESULT_NEED_SAMPLE_HOLDER will be returned once a sample has been reached.
Returns
  • One or more of the RESULT_* flags defined in this class.
Throws
ParserException If an error occurs parsing the media data.
