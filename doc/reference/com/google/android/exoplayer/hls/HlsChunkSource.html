public static final int ADAPTIVE_MODE_ABRUPT

Adaptive switches are performed at segment boundaries.

For this mode to perform seamless switches, the source content is required to have keyframes positioned at the start of each segment. If this is not the case a visual discontinuity may be experienced when switching from one variant to another.

Note that where it can be guaranteed that the source content does have keyframes positioned at the start of each segment, this mode should always be used in preference to ADAPTIVE_MODE_SPLICE because it requires fetching less data.

Constant Value: 3 (0x00000003)

public static final int ADAPTIVE_MODE_NONE

Adaptive switching is disabled.

The initially selected variant will be used throughout playback.

Constant Value: 0 (0x00000000)

public static final int ADAPTIVE_MODE_SPLICE

Adaptive switches splice overlapping segments of the old and new variants.

When performing a switch from one variant to another, overlapping segments will be requested from both the old and new variants. These segments will then be spliced together, allowing a seamless switch from one variant to another even if keyframes are misaligned or if keyframes are not positioned at the start of each segment.

Note that where it can be guaranteed that the source content has keyframes positioned at the start of each segment, ADAPTIVE_MODE_ABRUPT should always be used in preference to this mode.

Constant Value: 1 (0x00000001)

Parameters
dataSource A DataSource suitable for loading the media data.
playlistUrl The playlist URL.
playlist The hls playlist.
bandwidthMeter provides an estimate of the currently available bandwidth.
variantIndices A subset of variant indices to consider, or null to consider all of the variants in the master playlist.
adaptiveMode The mode for switching from one variant to another. One of ADAPTIVE_MODE_NONE, ADAPTIVE_MODE_ABRUPT and ADAPTIVE_MODE_SPLICE.
targetBufferSize The targeted buffer size in bytes. The buffer will not be filled more than one chunk beyond this amount of data.
targetBufferDurationMs The targeted duration of media to buffer ahead of the current playback position. The buffer will not be filled more than one chunk beyond this position.
minBufferDurationToSwitchUpMs The minimum duration of media that needs to be buffered for a switch to a higher quality variant to be considered.
maxBufferDurationToSwitchDownMs The maximum duration of media that needs to be buffered for a switch to a lower quality variant to be considered.
